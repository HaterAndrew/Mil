# ENVIRONMENT SETUP — EX-50N Advanced UI/UX Designer

**Track:** EX-50N — Advanced UI/UX Designer (TM-50N) | **Prerequisite:** TM-40N REQUIRED (Go evaluation on file) | **Base environment:** EX-40N
**Companion exams:** EXAM_TM50N_PRE (administer before exercise), EXAM_TM50N_POST (administer after exercise)

## Environment Type

Same as EX-40N: MSS with Workshop design access and external design tool (Figma or equivalent). See [EX-40N ENVIRONMENT_SETUP.md](../EX-40N_ux_designer/ENVIRONMENT_SETUP.md) for full base environment setup.

## Base Environment (EX-40N)

Complete **all** EX-40N environment setup steps before proceeding:

1. Design brief and requesting-unit scenario
2. Workshop environment with application creation permissions
3. Role-playing partners with guidance cards
4. Design tool access (Figma or equivalent) with MSS Design System Template
5. MSS accessibility checklist (TM-40N SS6-2) as printable handout

## TM-50N-Specific Additions

The following extend the EX-40N environment for advanced-level tasks.

### 1. Advanced Design Tools

| Tool | Purpose |
|------|---------|
| Accessibility testing tool | Automated WCAG AA/AAA compliance scanning (axe, Lighthouse, or equivalent) |
| Annotation/redlining tool | For DDIL tier annotations and classification marking overlays |
| Prototyping tool | Interactive prototype capability for multi-state designs (DDIL tier switching, cross-domain transitions) |

### 2. Multi-Application Portfolio

EX-50N requires trainees to evaluate and standardize across multiple applications. Provide:
- Read access to at least 3 existing Workshop applications (different teams, inconsistent styling)
- A portfolio inventory listing each application's purpose, primary users, and known UX inconsistencies

### 3. Cross-Domain Scenario Materials

For Task 3 (Cross-Domain UI Specification):
- Classification banner reference: DoD 5200.48 color/placement standards
- Sample multi-classification application mockup showing data from different classification levels
- ISSM review checklist template (or simulated equivalent for training)

### 4. Coalition Partner Reference Materials

For Task 4 (Coalition-Ready Interface):
- NATO standard abbreviation reference
- DTG and ISO 8601 date/time format reference card
- REL TO marking examples and releasability restriction guidance
- Metric/imperial dual-display requirements reference

### 5. Governance Template

For Task 5 (Governance Proposal):
- Example design review governance process from a comparable organization (template, not solution)
- Blank design quality metrics worksheet
