## What This Course Does for You

After this course you can independently access MSS, navigate to your unit's data products, consume and filter operational data, execute authorized status updates, and handle classification requirements for data exports. You will not be building anything — that starts at TM-20. This course is about operating what's already built.

Every Soldier on the data team completes TM-10 first, regardless of rank or technical background. There are no waivers.

---

## Learning Objectives

By the end of training, you will be able to:

1. Log in to MSS via CAC-based authentication and navigate to your unit's project
2. Locate, view, and filter data in a Workshop application
3. Execute an authorized Action (submit a status update, complete a form)
4. Build a basic chart and filter in Contour
5. Explore an Object Type in Quiver and export a view
6. Access and interact with an AIP Logic or Agent interface
7. Identify classification markings and state the authorized export procedure for a given dataset
8. Troubleshoot two common access issues using TM-10 procedures

## Practical Exercise

**Scenario:** You are an S3 NCO. Your unit's SITREP tracker and readiness dashboard are in MSS. You need to check for missing SITREP submissions, submit a correction, build a quick Contour chart, and export a filtered readiness table.

**Tasks evaluated:**
1. Login and navigate to the designated application
2. Filter table to last 7 days; identify missing submission
3. Execute the Status Update Action
4. Export filtered table to CSV
5. Build a bar chart in Contour from the provided dataset
6. State the classification marking and authorized distribution for the export

**Go standard:** Complete all 6 tasks; correctly identify the classification marking and export procedure.

---

## What "Go" Looks Like

You pass when an evaluator observes you completing all tasks independently — without asking for help, referring to notes beyond TM-10, or requiring hints. The Go standard is competency, not perfection. If you make a wrong click and recover yourself, that counts.

You do not pass by:
- Watching the instructor do it and saying you understand
- Completing tasks with the instructor talking you through each step

## Before You Attend: Access Requirements

You must have an active MSS account provisioned **before** you arrive. Contact your unit MSS Administrator at least **5 duty days** before training. Arriving without an account means you cannot complete the practical exercise.

Required: CAC reader on your workstation; network connectivity to MSS; account in the MSS Training Environment.

---

## Tips From Previous Graduates

- The login process (CAC + PIV PIN) trips up first-timers. Know your PIV PIN before you arrive.
- Classification questions on the practical exercise require you to read the dataset's marking fields — they are not always obvious. Check the dataset properties, not just the application header.
- AIP outputs are not authoritative. The interface is a tool, not an oracle. The practical exercise does not ask you to trust the AI — it asks you to use the interface correctly.
- If you get a "403 Forbidden" error, you have a permissions issue, not a login issue. State the correct resolution action (access request to MSS Administrator) — that is a tested task.
