## DAY 1 — UNDERSTANDING THE MATERIALS

### Block 1 — TM-10 Document Set Walkthrough
**Hours:** 1.5 | **Method:** Walkthrough | **Time:** 0830–1000

**Purpose:** The UDT delivers published materials, not self-designed instruction. This block ensures candidates know what every document is, where it lives, and how to use it.

**TLO:** The candidate will locate and describe the purpose of each TM-10 document (course content, lesson plans, syllabus, exercises, exams, T&EOs) and demonstrate how to use the lesson plan to deliver a block.

**Key Delivery Notes:**
- Open each document in the repository and walk through its structure
- Focus on the lesson plans: show the block structure (Purpose, TLO/ELOs, Instructor Notes, Student Activity, Check-on-Learning, Common Errors)
- Emphasize: "Follow the lesson plan. If you think something is wrong, deliver as written and report to C2DAO after."
- Show the T&EO scoring sheets: explain each performance measure column and the critical item markers

**Student Activity:** Each candidate opens each document independently; confirms they can navigate from lesson plan → exercise → T&EO for Block 2 of TM-10.

**Assessment:** Formative — all candidates must independently locate all documents.

---

### Block 2 — TM-20 Refresher Materials (Blocks 1-10)
**Hours:** 1.25 | **Method:** Walkthrough | **Time:** 1015–1130

**Purpose:** UDTs can facilitate TM-20 refresher labs for Blocks 1-10 only. This block defines the boundary.

**TLO:** The candidate will identify which TM-20 blocks are within UDT refresher scope (1-10) and which require a C2DAO instructor (11+).

**Key Delivery Notes:**
- Blocks 1-10 scope: project creation, file ingestion, single-source pipeline basics
- Blocks 11+ out of scope: multi-source joins, computed columns, Workshop, Ontology, governance workflow
- Walk through Blocks 1-10 lesson plan overview — candidates do not need to memorize, but need to know what topics are covered
- Emphasize: "Refresher only. You cannot deliver initial TM-20 certification."

**Student Activity:** Sort a provided list of 15 TM-20 topics into "in scope" and "out of scope" for UDT refresher facilitation.

**Common Errors:**
- Candidate assumes all of TM-20 is in scope — reinforce the Blocks 1-10 boundary
- Candidate confuses "refresher" with "initial certification" — refresher is for TM-20-qualified personnel who need practice, not for new students

**Assessment:** Formative — sorting exercise must be 100% correct (hard boundary, no ambiguity).

### Block 3 — Abbreviated Adult Learning Principles
**Hours:** 0.5 | **Method:** Seminar | **Time:** 1130–1200

**Purpose:** UDTs need three principles, not the full adult learning theory from T3-I.

**TLO:** The candidate will explain the three UDT-relevant principles: (1) adults learn by doing, (2) adults need to know why, (3) productive vs. unproductive errors.

**Key Delivery Notes:**
- Principle 1: Lab-dominant instruction. TM-10 is mostly hands-on. Let the trainees work.
- Principle 2: "Why first." Before every block, say why this matters to the trainee's job.
- Principle 3: Productive errors (wrong join type → learning moment) vs. unproductive errors (can't log in → infrastructure failure you should have prevented)

**Student Activity:** For each of the 3 principles, candidates write 1 example of applying it to a TM-10 block.

**Assessment:** Formative.

---

### Block 4 — Pre-Course Checklist Execution
**Hours:** 1.5 | **Method:** Lab | **Time:** 1300–1430

**Purpose:** A failed pre-course setup wastes half of a 1-day TM-10 course.

**TLO:** The candidate will execute the UDT pre-course checklist for a simulated TM-10 iteration, including account verification, environment testing, and materials preparation.

**Key Delivery Notes:**
- Walk through UDT SOP §4-4 pre-course checklist step by step
- Candidates execute each step on a simulated 8-student class roster
- Plant 1 issue: one student account at Viewer access instead of the required level — candidate must identify and escalate
- Print preparation: demonstrate printing student handouts, exams (student version only), T&EO scoring sheets
- Answer key security: demonstrate secure storage in instructor binder

**Student Activity:** Execute the full pre-course checklist. Identify the planted access issue.

**Assessment:** Formative — planted issue must be identified.

---

### Block 5 — Exam Administration
**Hours:** 1.25 | **Method:** Lab | **Time:** 1445–1600

**Purpose:** UDTs administer TM-10 pre-tests, post-tests, and standalone exam proctoring. Correct procedure protects evaluation integrity.

**TLO:** The candidate will administer a mock TM-10 pre-test, score it using the answer key, and document results — following standard procedures for timing, key security, and score recording.

**Key Delivery Notes:**
- Pre-test: Day 1 morning, 20 minutes, diagnostic (does not count toward Go/No-Go)
- Post-test: end of course, 30 minutes, passing score per POI
- Standalone proctoring: for self-study/remote personnel; same post-test conditions; coordinate with C2DAO for practical exercise scheduling after passing
- Answer key: never distributed; print student versions only; secure key in instructor binder
- Scoring: use answer key exactly; record individual scores; compute class mean

**Student Activity:** Administer a mock pre-test to role-players (5 minutes). Score using answer key. Record results. Practice securing the answer key.

**Assessment:** Formative — procedure compliance reviewed.

### Block 6 — Reporting Requirements
**Hours:** 1.0 | **Method:** Seminar | **Time:** 1600–1700

**Purpose:** UDTs report training status to C2DAO. Without reports, C2DAO has no visibility into unit-level training.

**TLO:** The candidate will complete a Unit Training Status Report in the correct format and explain the No-Go reporting timeline (24 hours).

**Key Delivery Notes:**
- Walk through UDT SOP §5-1 report format: all fields, submission timeline (5 duty days), submission method
- No-Go reporting: 24 hours, to C2DAO distribution list
- Escalation reporting: unresolved issues within 48 hours
- Show a completed example report
- TM-20 refresher reporting: informal email format

**Student Activity:** Complete a Unit Training Status Report using sample data from a simulated TM-10 iteration (8 students, 1 No-Go, 2 environment issues).

**Assessment:** Formative — report completeness reviewed.

## DAY 2 — EVALUATION AND TROUBLESHOOTING

### Block 7 — T&EO Walkthrough (TM10-01 through TM10-06)
**Hours:** 2.0 | **Method:** Lab | **Time:** 0830–1030

**Purpose:** UDTs must understand every TM-10 T&EO well enough to score accurately and consistently.

**TLO:** The candidate will walk through TM10-01 through TM10-06, identifying all performance measures and all critical items, and explain how the overall Go/No-Go is determined.

**Key Delivery Notes:**
- Walk through each T&EO sequentially: task, condition, standard, performance measures, critical items
- Identify critical items: classification marking (TM10-06), production environment avoidance
- Explain the Go determination: Go on all critical items + Go on required number of non-critical items
- Trace from T&EO back to the TM-10 lesson plan: "This T&EO evaluates what you taught in Block X"

**Student Activity:** For each T&EO, candidates mark which performance measures are critical. Compare with the answer. Discuss any discrepancies.

**Assessment:** Formative — all critical items must be correctly identified.

---

### Block 8 — Go/No-Go Decision Exercise
**Hours:** 1.25 | **Method:** Lab | **Time:** 1045–1200

**Purpose:** Borderline cases are where evaluation consistency breaks down. UDTs must be calibrated.

**TLO:** Given 4 borderline trainee performances, the candidate will score each using the T&EO and make the correct Go/No-Go decision, reaching group consensus.

**Key Delivery Notes:**
- 4 scenarios (role-played or described):
  1. Trainee completes all tasks but uses non-standard navigation method (Go — result meets standard)
  2. Trainee completes all tasks but applies wrong classification marking on export (No-Go — critical item failed)
  3. Trainee asks instructor a clarifying question during evaluation (depends on the question — "what does this button do?" = assistance = void evaluation; "can I use my notes?" = procedural question = OK)
  4. Trainee completes 4 of 5 non-critical items, all critical items passed (Go or No-Go depends on T&EO threshold — look it up)
- Each candidate scores independently before group discussion

**Student Activity:** Score all 4 scenarios. Group calibration discussion. Document reasoning.

**Assessment:** Formative — correct overall decision on at least 3 of 4 required.

---

### Block 9 — No-Go Procedures
**Hours:** 1.5 | **Method:** Lab | **Time:** 1300–1430

**Purpose:** Delivering a No-Go to someone from your own unit is difficult. This block practices the procedure and the interpersonal skill.

**TLO:** The candidate will conduct a simulated No-Go counseling session that identifies specific deficiencies, documents the result, and coordinates remediation.

**Key Delivery Notes:**
- No-Go counseling structure: immediate (same-day), specific (which performance measures), documented (T&EO scoring sheet + written note), constructive (remediation plan)
- Role-play: instructor plays a disappointed trainee. Candidate must maintain professionalism, specificity, and constructive tone.
- Documentation practice: complete the T&EO scoring sheet for the No-Go scenario; draft a remediation note
- Coordinate: with unit training NCO for remediation scheduling; report to C2DAO within 24 hours

**Student Activity:** Each candidate conducts a simulated No-Go counseling. Peer and instructor feedback.

**Assessment:** Formative — instructor reviews documentation and counseling delivery.

---

### Block 10 — Environment Troubleshooting
**Hours:** 1.25 | **Method:** Lab | **Time:** 1445–1600

**Purpose:** UDTs must resolve the 5 most common issues without C2DAO support.

**TLO:** The candidate will diagnose and resolve 5 pre-staged environment failures within the prescribed time limits.

**Key Delivery Notes:**
- 5 pre-staged failures: CAC auth (5 min), expired account (10 min), wrong environment (2 min), missing project access (10 min), stale training data (5 min)
- Each candidate rotates through all 5 stations
- Debrief each: root cause, resolution, prevention, when to escalate

**Student Activity:** Rotate through 5 troubleshooting stations. Resolve each within the time limit.

**Assessment:** Formative — all 5 should be resolved. Failures debriefed and re-attempted.

---

### Block 11 — Escalation Decision Matrix
**Hours:** 1.0 | **Method:** Seminar | **Time:** 1600–1700

**Purpose:** Knowing when to escalate is the most important UDT skill. Attempting to handle training beyond UDT scope causes more harm than delaying until C2DAO is available.

**TLO:** The candidate will correctly classify 6 provided scenarios as "handle locally" or "escalate to C2DAO" and explain the escalation procedure.

**Key Delivery Notes:**
- Walk through the escalation matrix from Concepts Guide §4-1
- 6 scenarios for classification:
  1. Student can't log in — handle locally (CAC auth troubleshooting)
  2. Commander asks for TM-20 delivery — escalate (beyond UDT scope)
  3. Student fails re-evaluation — escalate (C2DAO guidance needed)
  4. Stale training data — handle locally if loader available; escalate if not
  5. Student visible to production data from training account — escalate immediately (security concern)
  6. Lesson plan appears to have an error — deliver as written, report after (escalate post-delivery)
- 48-hour reporting timeline for unresolved issues

**Student Activity:** Classify all 6 scenarios. Discuss. Resolve any disagreements.

**Assessment:** Formative — all 6 must be correctly classified.

## DAY 3 — TEACH-BACK AND EVALUATION

### Block 12 — Teach-Back (Evaluated)
**Hours:** 3.0 | **Method:** Evaluation | **Time:** 0830–1130

**Purpose:** The teach-back is the primary performance assessment for T3-F.

**TLO:** The candidate will deliver a 15–20 minute TM-10 block demonstrating satisfactory technical accuracy and materials use.

**Key Delivery Notes:**
- Each candidate delivers 1 TM-10 block selected during Day 2 evening preparation
- Other candidates role-play as TM-10 trainees
- Evaluator scores on: (1) Technical accuracy — content is correct; (2) Materials use — follows lesson plan, uses check-on-learning questions
- Schedule 20 minutes per candidate (15-20 delivery + debrief); 8 candidates = 2.7 hours

**Student Activity:** Deliver the prepared block. Receive feedback.

**Assessment:** **EVALUATED.** Go: satisfactory on both technical accuracy and materials use. No-Go: unsatisfactory on either.

---

### Block 13 — Go/No-Go Practicum (Evaluated)
**Hours:** 1.0 | **Method:** Evaluation | **Time:** 1300–1400

**Purpose:** Verifies the candidate can administer a T&EO correctly under observation.

**TLO:** The candidate will administer 1 T&EO to a role-player, making the correct overall Go/No-Go decision with proper documentation and mode transition.

**Key Delivery Notes:**
- Instructor assigns a T&EO from TM10-01 through TM10-06
- Role-player performs a predetermined scenario (Go or No-Go)
- Evaluator observes: correct scoring of all performance measures, correct overall decision, clear mode transition announcement, no assistance during evaluation, proper documentation
- Each candidate gets a different T&EO to prevent pattern sharing

**Student Activity:** Administer the assigned T&EO. Score. Document. Announce mode transitions.

**Assessment:** **EVALUATED.** Go: correct overall decision + proper documentation + no assistance provided. No-Go: incorrect decision, assistance provided, or missing documentation.

### Block 14 — Troubleshooting Exam (Evaluated)
**Hours:** 0.5 | **Method:** Evaluation | **Time:** 1400–1430

**Purpose:** Verifies the candidate can resolve common environment issues independently.

**TLO:** The candidate will resolve 2 of 3 pre-staged environment failures within prescribed time limits.

**Key Delivery Notes:**
- 3 issues drawn from the set of 5 common failures (different from Day 2 practice set where possible)
- Each candidate works independently
- Time limits per issue type apply
- Evaluator confirms resolution

**Student Activity:** Diagnose and resolve 3 presented issues.

**Assessment:** **EVALUATED.** Go: 2 of 3 resolved within time limits. No-Go: fewer than 2 resolved.

### Block 15 — AAR
**Hours:** 1.0 | **Method:** Discussion | **Time:** 1500–1600

**Purpose:** Structured review of T3-F iteration; Go/No-Go results; what to do when you return to your unit.

**Key Delivery Notes:**
- Standard AAR format
- Go/No-Go results delivered individually
- For Go candidates: "Here is what to do first when you get back to your unit" — schedule first TM-10, coordinate with training NCO, notify C2DAO
- For No-Go candidates: specific remediation path and timeline

**Student Activity:** Participate in AAR. Provide feedback.

### Block 16 — Certification and Closeout
**Hours:** 1.0 | **Method:** Brief | **Time:** 1600–1700

**Purpose:** Administrative closeout; UDT designation paperwork; sustainment requirements.

**Key Delivery Notes:**

**Student Activity:** Complete designation paperwork. Receive templates and contact information.
