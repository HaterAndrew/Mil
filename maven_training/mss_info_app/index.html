1. WHAT IS MSS?

MSS is a secure, web-based platform where your unit’s data lives and can be analyzed and acted upon. Think of it like a shared operations center for data: information from logistics, personnel, readiness, and other systems is collected, organized, and made accessible through applications your unit uses every day.

MSS is built on the Palantir Foundry platform, authorized for Army use under the Maven Smart System program.

MSS DOES
  • Stores data from Army systems in a single, organized location
  • Makes data visible through applications
  • Enables units to update records, report status, and track readiness
  • Provides analysis tools for authorized personnel
  • Supports AI-assisted analysis through authorized tools
MSS IS NOT
  • Not a replacement for official Army systems of record (DCPDS, GCSS-A, MEDPROS, etc.)
  • Not classified by default — classification depends on data and markings
  • Not a public system — access is controlled and audited
USAREUR-AF MISSION AREAS
  • Personnel Readiness — soldier readiness status
  • Logistics — equipment availability & maintenance
  • Operational Reporting — SITREPs and updates
  • Planning — orders, unit positions, task org
  • C2 — unit status across the AOR

2. SECURITY RESPONSIBILITIES

WARNING
Unauthorized access to, disclosure of, or modification of data in MSS may constitute a violation of 18 U.S.C. § 1030 (Computer Fraud and Abuse Act) and applicable Army regulations. Violations may result in disciplinary action, loss of access, and criminal prosecution.
  1. Use only your own credentials. Do not share your CAC, PIN, or any access tokens.
  2. Access only data you are authorized to view.
  3. Report misrouted data immediately. If you see data at a higher classification than your clearance — stop and report it. Do not read, copy, or act on that data.
  4. Do not export data without authorization. Exports are logged.
  5. Log out when done. Do not leave an MSS session unattended on an unlocked workstation.
  6. Report incidents immediately to your supervisor and unit security officer.

3. TASKS

TASK: ACCESS THE MAVEN SMART SYSTEM
Conditions
Provisioned MSS account, CAC reader, approved workstation and browser
Standards
Successfully authenticate with CAC and reach the MSS home screen
Equipment
CAC, CAC reader, workstation, MSS portal URL (from unit S6)
  1. Insert your CAC into the CAC reader.
  2. Open an approved web browser (Chrome or Firefox recommended).
  3. Navigate to the MSS portal URL provided by your unit S6.
  4. When prompted, select your authentication certificate (not your email certificate).
  5. Enter your CAC PIN when prompted.
  6. MSS home screen loads — you are now logged in.
CAUTION
Do not save your PIN in the browser. Do not allow the browser to remember your login. MSS sessions may contain sensitive information.

4. GETTING ACCESS

  1. Contact your unit S6 or data steward to initiate an account request.
  2. Supervisor submits request with your name, unit, role, and required access level.
  3. MSS admin team provisions your account and assigns markings.
  4. You receive notification when account is active.
  5. Log in via the MSS portal URL provided by your unit.
NOTE — PROVISIONING TIME
Account provisioning may take 3–5 business days. Plan accordingly before a deployment or exercise requiring MSS access.

BASIC TRANSFORM SKELETON

from transforms.api import transform_df, Input, Output
import pyspark.sql.functions as F

@transform_df(
    Output("/Project/AOR/source/curated/my_dataset"),
    source=Input("/Project/AOR/source/staging/my_dataset"),
)
def compute(source):
    return (
        source
        .filter(F.col("required_field").isNotNull())
        .dropDuplicates(["id"])
        .withColumn("processed_dt", F.current_timestamp())
    )

INCREMENTAL PIPELINE PATTERN

from transforms.api import incremental, Input, Output
import pyspark.sql.functions as F

@incremental(
    require_incremental=True,
    semantic_version=1,
)
@transform_df(
    Output("/Project/AOR/source/curated/events_incremental"),
    source=Input("/Project/AOR/source/staging/events"),
)
def compute(source, ctx):
    # Only process new data since last run
    if ctx.is_incremental:
        source = source.filter(
            F.col("event_dt") > ctx.last_run_input_watermark
        )
    return source.dropDuplicates(["event_id"])

COMMON PYSPARK OPERATIONS

# Filter rows
df.filter(F.col("status") == "ACTIVE")

# Add / transform column
df.withColumn("unit_upper", F.upper(F.col("unit_name")))

# Rename column
df.withColumnRenamed("old_col", "new_col")

# Remove duplicates
df.dropDuplicates(["id"])

# Join datasets
df1.join(df2, "unit_id", "left")

# Group and count
df.groupBy("unit").agg(F.count("*").alias("record_count"))

# Cast data type
df.withColumn("count", F.col("count").cast("integer"))

# Fill nulls
df.fillna({"status": "UNKNOWN"})

# Current timestamp
F.current_timestamp()

# Parse timestamp string
F.to_timestamp(F.col("dtg_str"), "yyyy-MM-dd HH:mm:ss")

CORE DATA LITERACY CONCEPTS (ADRP 1 SUMMARY)

DATA TYPES
Structured (tables, rows, columns) • Semi-structured (JSON, XML) • Unstructured (documents, images). MSS ingests all three types from Army source systems.
DATA PIPELINE
Raw → Staging → Curated → Ontology → Application. Each layer adds quality, structure, and meaning. Never modify raw data — it is the source of truth.
DATA QUALITY
Accuracy, completeness, consistency, timeliness, uniqueness. Bad decisions follow bad data. Report quality issues to your data steward rather than working around them.
DATA GOVERNANCE
Ownership, stewardship, access control, classification markings. Every dataset has an owner. Every user has a role. Access is audited. Misuse is tracked.
